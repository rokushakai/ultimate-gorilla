====================================================
究極ゴリラ ULTIMATE GORILLA (試作版 v2)
====================================================

究極のUMA「究極ゴリラ」を探す勇者の子孫たちの旅。
HTML / CSS / JavaScript のみで作られたブラウザRPGです。

このゲームは「UMA収集RPG」です。育つのは主人公だけで、
UMAは育成せず、捕まえて図鑑に登録する・売ってお金にする対象として
扱っています(ポケモンのように仲間にして戦わせる育成要素はありません)。

Node.js や Python、サーバーの起動は一切不要です。
インターネット接続も不要です(完全オフラインで動作します)。


----------------------------------------------------
■ 起動方法
----------------------------------------------------
1. このフォルダを好きな場所に展開(解凍)してください。
2. フォルダの中にある index.html をダブルクリックしてください。
   → 既定のブラウザ(Google Chrome や Microsoft Edge)が開き、
     そのままゲームが起動します。

  ※ ファイル構成は以下の通りです。すべて同じフォルダに
    入っている必要があります。
      index.html  … ゲーム画面の本体
      style.css   … 見た目のスタイル
      script.js   … ゲームのロジック・データ(プログラム本体)
      README.txt  … この説明書

  ※ 推奨ブラウザ: 最新版の Google Chrome または Microsoft Edge
    (Windows 11 に標準で入っている Edge でそのまま動作します)

  ※ スマホで遊びたい場合は、このフォルダをZIPにしてスマホに
    送り、解凍してから index.html をブラウザで開いてください。
    (Android の Chrome、iPhone の Safari どちらでも動作します)


----------------------------------------------------
■ 配布方法(ZIP化)
----------------------------------------------------
このフォルダ全体(index.html / style.css / script.js / README.txt)を
右クリック→「送る」→「圧縮(zip形式)フォルダー」などでZIP化し、
相手に渡してください。相手は解凍して index.html を開くだけで遊べます。
インストール作業は不要です。

  ※ backup_v1 フォルダは旧バージョンの控えです。配布時に含めても
    問題はありませんが、不要であれば削除しても構いません。


----------------------------------------------------
■ 遊び方
----------------------------------------------------
[フィールド画面]
・画面上のマップを移動します。
  - スマホ: 画面下の十字キー(▲▼◀▶)をタップ、または
            フィールド画面を上下左右にスワイプ
  - PC:    キーボードの矢印キーでも移動できます(動作確認用)
・草原(緑のタイル 🟩)を歩くと、一定確率でモンスターや
  UMA(未確認生物)に遭遇します。
  茶色の道(🟫)や村(🏠)、商人(🏪)、神社(⛩️)の上では遭遇しません。
・マップに落ちている 🗡️(武器)に触れると自動で拾い、
  こうげき力が上がります。
  🧪(やくそう)に触れると自動で拾い、戦闘中に使えます。

[戦闘画面]
敵に遭遇すると戦闘になります。コマンドは5つ。
  ⚔ たたかう   … 武器で攻撃する
  ✨ まほう     … 覚えたまほうを使う(MPを消費)
  🧪 アイテム   … やくそう(回復)や捕獲ロープ(捕獲率UP)を使う
  🪤 つかまえる … 敵を捕獲する。HPが減っているほど成功しやすい
  💨 にげる     … 戦闘から逃げる

・倒す、逃げる、捕まえる、いずれの結果でも経験値が手に入ります
  (逃げた場合は少なめです)。
・経験値が一定値を超えるとレベルアップします。
・レベルアップ時、ランダムで新しい「まほう」を1つ覚えます。
  (HP/MPの上限やこうげき力・ぼうぎょ力も上がります)
・HPが0になっても村に戻ってやり直せます(ペナルティは軽めです)。

[UMA図鑑]
・画面右上の「📖図鑑」ボタンから、これまでに遭遇したUMAの
  一覧を確認できます。3段階で表示されます。
    ？？？        … 未発見(一度も遭遇していない)
    UMA名(発見済) … 戦闘で遭遇したが、まだ捕獲していない
    UMA名(捕獲済) … 一度でも捕獲に成功した
  捕獲済みのUMAを後で売っても、図鑑の「捕獲済み」記録は消えません。

[レアUMAと最終目標]
・ビッグフット、ネッシー、イエティ、ジャージーデビルなどの
  レアUMAは、レベルに関係なくどこでも低確率で出現します。
  かなり強いので、勝てそうにない時は「にげる」を選んでください
  (レアUMAは比較的逃げやすく調整されています)。
・そして、ごく低確率で伝説のUMA「究極ゴリラ」🦍 が出現します。
  これを発見することが、このゲームの最終目標です。
  発見した瞬間に専用のメッセージが表示されます。
  (倒せなくても、発見した時点で目標達成という位置づけです)

[究極ゴリラについて(重要)]
・究極ゴリラは完全にラスボス級の強さで、レベルが低いうちはまず
  間違いなく一撃で倒されます。攻撃はすべて「かいしんのいちげき」級の
  大ダメージで、ぼうぎょ力で受け止めることはほぼできません。
・倒す・捕まえることは、レベル99かつ最強装備クラスでもほぼ不可能な
  難易度にしてあります(理論上ゼロではありませんが、通常プレイでは
  現実的に勝てません)。捕獲も成功率に特別な上限がかかっています。
・低レベルでも出会うこと自体は普通にあります。出会ったら戦う必要は
  なく、「にげる」を選べばほぼ確実に逃げられます(究極ゴリラは
  あえて逃げやすく調整されています)。倒れてしまっても村に戻って
  やり直せるので、安心して遭遇を楽しんでください。


----------------------------------------------------
■ 商人の使い方(フィールドの🏪に触れる)
----------------------------------------------------
村の中にある 🏪(商人)に歩いて触れると、売買メニューが開きます。
  🛍️ 買う          … 鉄の剣・やくそう・捕獲ロープを所持金で購入
  📤 アイテムを売る … 持っているやくそう/捕獲ロープを売ってお金にする
  🦍 UMAを売る      … 捕獲して所持しているUMAを売ってお金にする
  やめる            … 商人メニューを閉じる

・買い物のお金は画面左上の「💰 ○○G」(所持金)で確認できます。
・鉄の剣を買うと、その場でこうげき力が上がります
 (装備の付け外しはなく、買った瞬間に効果が反映される簡易仕様です)。
・お金が足りない時は購入できません(トーストで知らせます)。

[UMA売却について]
・「UMAを売る」では、今持っているUMAの種類と数、売却額が一覧表示
  されます。売ると所持数が1減り、お金が増えます。
・同じUMAを複数捕まえていれば、何匹でも売れます。
・UMAを売り切っても、UMA図鑑の「捕獲済み」記録はそのまま残ります
 (図鑑は「捕まえた実績」、所持UMAは「今持っている数」という扱いです)。


----------------------------------------------------
■ 転職の説明(フィールドの⛩️に触れる)
----------------------------------------------------
村の中にある ⛩️(神様の社)に歩いて触れると、転職メニューが開きます。
なりたい部活(職業)のボタンを選ぶと、その場で転職します。
お金や条件は不要で、いつでも何度でも転職し直せます。
現在の職業は画面左上のステータス欄に「(○○部)」と表示されます。

[職業ごとの特徴]
  野球部     … こうげき力が少し高い
  水泳部     … 最大HPが少し高い
  テニス部   … UMAを捕まえやすい
  帰宅部     … 戦闘から逃げやすい
  吹奏楽部   … まほうを覚えやすい(レベルアップ時に追加でまほうを覚えることがある)
  サッカー部 … 全部少しずつ強い、バランス型(最初の職業)
  ラグビー部 … HPとこうげき力が高いが、その分逃げにくい
  陸上部     … とても逃げやすい
  魔法戦士   … まほうとこうげきのバランス型(MPも少し増える)

・転職すると、こうげき力・ぼうぎょ力・最大HP・最大MPがその場で
  再計算されます。最大値が下がる転職をした場合、現在のHP/MPが
  超えていれば新しい上限まで自動的に下がります(全回復はしません)。
・捕獲率/逃走率/まほうの覚えやすさへの補正は、戦闘中の判定にだけ
  反映されます(見た目のステータス欄には数値で出ません)。

■ 設定画面(画面右上「⚙️設定」)
----------------------------------------------------
画面右上の「⚙️設定」ボタンから設定画面を開けます。
現時点では「主人公の歩く速さ」だけを変更できます。

  遅い / 普通(初期値) / 速い

・十字キーを押しっぱなしにすると、選んだ速さで連続移動します。
  軽くタップした場合は、これまでと同じく1回のタップで1歩だけ
  動きます(タップ操作の感覚は変わりません)。
・設定はセーブデータと一緒にブラウザに保存されます。


----------------------------------------------------
■ セーブ/ロードについて
----------------------------------------------------
ブラウザの localStorage を使って自動セーブしています。
アイテム取得・戦闘終了・買い物・転職などのたびに自動で保存され、
次回 index.html を開いたときに自動で読み込まれます
(読み込まれた場合は「💾 前回のデータを読み込みました」と表示されます)。

  保存される内容: レベル/経験値/HP・MP/覚えたまほう/所持金/
  所持アイテム数/所持UMA/UMA図鑑の状態/現在の職業

  保存されない内容(TODO): フィールド上の現在位置、拾った/拾って
  いないフィールドアイテムの状態。再読み込み時は村の入口から
  スタートします。必要であれば script.js の saveGame()/loadGame()
  に state.player.x, y や state.items の差分を追加してください。

  ※ localStorageはブラウザ・端末ごとに保存されるため、別のPCや
    別のブラウザ、別の場所に展開したフォルダには引き継がれません。
  ※ ブラウザの設定でサイトデータ/履歴を消去すると、セーブデータも
    一緒に消えます。


----------------------------------------------------
■ 開発者向けメモ: データの編集場所
----------------------------------------------------
※ ゲーム仕様の正式なドキュメントは GAME_DESIGN.md、実装予定は TODO.md、
  変更履歴は CHANGELOG.md にまとめてあります。新しい仕様はまず
  GAME_DESIGN.md に整理してから実装する運用にしています。

あばれうまのりお からのアイデア追加など、今後のネタ出しに対応
しやすいよう、script.js の冒頭(「2. データ定義」のコメント以降)に
データをまとめています。基本的にロジック部分(関数)を触らずに、
配列へ要素を追加・編集するだけで内容を拡張できます。

  UMA_DATA      … 捕獲・図鑑登録・売却の対象になるUMA本体のデータ
                   (id/name/rarity/hp/attack/captureRate/exp/
                    sellPrice/isRareなど)
  NON_UMA_DATA  … UMAではない雑魚モンスター(スライム等。図鑑対象外)
  ITEM_DATA     … やくそう・捕獲ロープなどの消耗品(価格・効果)
  WEAPON_DATA   … 武器(こうげき力の加算値・購入価格)
  SPELL_DATA    … まほう(消費MP・威力・攻撃/回復タイプ)
  JOB_DATA      … 職業(部活)ごとのステータス補正と説明文

例えば新しいUMAを1匹追加したい場合は、UMA_DATA配列に1行追加する
だけで、図鑑・エンカウント・捕獲・売却すべてに自動的に反映されます。

究極ゴリラの出現を素早く確認したい場合は、script.js の
「3. エンカウント関連の調整値」付近にある下記の定数を調整してください。

  var RARE_ENCOUNTER_CHANCE = 0.08;  // ここを上げるとレアUMA全体が出やすくなる

さらに UMA_DATA 配列の中の
  { id: "ultimategorilla", ... weight: 4 ... }
の weight の値を大きくすると、レアUMAの中でも究極ゴリラが
選ばれやすくなります。

マップ(RAW_MAP)を編集して、商人(M)や神様(G)の位置を増やしたり、
新しいエリアを追加することもできます。1行=1文字につき1マスです。

■ 将来の課金要素メモ(今回は未実装)
----------------------------------------------------
今回のプロトタイプには課金処理は一切入れていません。
将来アプリストア等で配信する場合に向けたメモとして書き残します。

・倒れた(全滅した)時の「コンティニュー」を課金制にできる設計を
  想定しておく(例: 無料では村に戻るだけ、課金すると戦闘中のHPを
  維持して再戦できる、など)。
・実装する場合は、現在 handlePlayerDown() で行っている「村に戻る」
  処理の前段に、課金導線を差し込めるような分岐を用意すると良い。
・ストア審査やプラットフォームの決済API対応が必要になるため、
  実際の課金処理はWebブラウザの素のJavaScriptだけでは完結しない
  (ネイティブアプリ化やストア決済SDKの導入が前提になる)。
