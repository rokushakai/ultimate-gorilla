# 究極ゴリラ ULTIMATE GORILLA

究極のUMA「究極ゴリラ」を探す勇者の子孫たちの旅。
**スマホ1画面で延々と経験値を稼ぐ、UMA収集ブラウザRPG**です。

HTML / CSS / JavaScript のみで作られており、外部ライブラリやビルド、
サーバーの起動は一切不要です。ブラウザで `index.html` を開くだけで
そのまま動作します。

🎮 **公開URL（GitHub Pages）:** https://rokushakai.github.io/ultimate-gorilla/
*( 公開直後はビルドに数分かかることがあります )*


## 概要

- 育つのは主人公だけ。UMA は **育成せず、捕まえて図鑑に登録する・売る対象**として扱う収集RPGです。
- フィールドを歩き、ランダムエンカウントで戦闘 → 経験値でレベルアップ → まほうを習得、というループを1画面の中で繰り返します。
- 商人で武器やアイテムを売買し、神社で職業（部活）を変えて戦い方を調整できます。
- 最終目標は、ラスボス級の伝説のUMA「究極ゴリラ」🦍 を**発見すること**です。


## 遊び方

1. フィールド画面で草原（緑のタイル）を歩くと、一定確率でモンスターやUMAに遭遇します。
2. 戦闘では「⚔ たたかう」「✨ まほう」「🧪 アイテム」「🪤 つかまえる」「💨 にげる」から選びます。
3. 倒す・逃げる・捕まえる、いずれの結果でも経験値が入りレベルアップします。
4. レベルアップ時にランダムで新しい「まほう」を1つ習得します。
5. 村にある 🏪 商人で武器/やくそう/捕獲ロープを売買し、UMAを売ってお金にできます。
6. 村にある ⛩️ 神社で職業（部活）を変更でき、ステータスや得意分野が変わります。
7. レアUMA（ビッグフット・ネッシー・イエティ・ジャージーデビルなど）はレベルに関係なく低確率で出現します。強いので、勝てない時は「にげる」を選びましょう。
8. ごく低確率で伝説のUMA「究極ゴリラ」が出現します。これを発見することがゲームの最終目標です。
   **究極ゴリラはラスボス級で、レベル99・最強装備クラスでもほぼ倒せません。** 出会ったら「にげる」でほぼ確実に逃げられます。安心して発見を楽しんでください。

詳しい仕様や開発メモは [README.txt](README.txt) にまとめています。


## 操作方法

| 操作 | 内容 |
|---|---|
| 画面下の十字キー（▲▼◀▶） | タップで1歩移動。押しっぱなしで連続移動（速度は設定で変更可） |
| フィールドのスワイプ | 上下左右にスワイプして移動（スマホ向け） |
| キーボードの矢印キー | PCでの動作確認用 |
| 画面右上「📖図鑑」 | これまでに遭遇/捕獲したUMAの一覧を表示 |
| 画面右上「⚙️設定」 | 主人公の歩く速さ（遅い/普通/速い）を変更 |

スマホ縦画面を前提にしていますが、PCブラウザでも幅480px程度の
中央レイアウトで見やすく表示されます。


## ZIPで配布する方法

Node.js / Python なしで、`index.html` をダブルクリックするだけで
動作します（完全オフライン対応）。

1. このリポジトリを `Code → Download ZIP` でダウンロード、または
   `git clone` してフォルダを取得します。
2. フォルダ全体（`index.html` / `style.css` / `script.js` / それ以外のREADME等）を
   右クリック → 「送る」→「圧縮(zip形式)フォルダー」などでZIP化します。
3. 相手はZIPを解凍して `index.html` を開くだけで遊べます。
   インストール作業やインターネット接続は不要です。


## GitHub Pages 公開手順

このリポジトリでは既にGitHub Pagesを有効化済みです
（公開URLは冒頭に記載）。同じ設定をGitHub上のUIから確認・変更したい場合は、
リポジトリの **Settings → Pages** を開き、以下の内容になっていることを確認してください。

1. `Settings` タブを開く
2. 左メニューの `Pages` を開く
3. `Source` が `Deploy from a branch` になっている
4. `Branch` が `main` になっている
5. `Folder` が `/ (root)` になっている
6. （変更した場合は）`Save` する
7. 数分後、ページ上部に公開URL（`https://<ユーザー名>.github.io/ultimate-gorilla/`）が表示されます

別のアカウント/リポジトリで初めて公開する場合は、上記を新規に設定し、
公開後にこのREADME.md冒頭の「公開URL」欄を実際のURLに書き換えてください。

`index.html` はリポジトリ直下にあり、`style.css` / `script.js` は
すべて相対パスで読み込んでいるため、サブディレクトリでホストされる
GitHub Pages でもそのまま動作します。


## GitHubへのアップロード手順（参考: 別環境にリポジトリを作り直す場合）

このリポジトリは既に `https://github.com/rokushakai/ultimate-gorilla` に
push済みで、GitHub Pagesも有効化されています。別のPCやアカウントで
同じ手順を再現したい場合は、以下のコマンドで作成できます。

```bash
git init
git add .
git commit -m "Initial playable prototype"
git branch -M main
git remote add origin https://github.com/<ユーザー名>/ultimate-gorilla.git
git push -u origin main
```

※ 初回pushでGitHubの認証（ブラウザでのログイン、または
  Personal Access Token の入力）を求められる場合があります。
  画面の指示に従ってください。
※ `gh` CLI（GitHub CLI）でログイン済みであれば、
  `gh repo create ultimate-gorilla --public --source=. --remote=origin` で
  リポジトリ作成とリモート設定を一度に行うこともできます。


## 今後追加予定の機能

- 複数の街・ダンジョン・ワールドマップ間の移動（現在は1枚のマップのみ）

- セーブデータにプレイヤー座標やフィールドアイテムの取得状況も含める
- 図鑑からUMAの詳細ステータスを見られるようにする
- **(将来メモ)** ストア配信する場合、全滅時の「コンティニュー」を課金制にできる設計を想定。
  現時点では課金処理は未実装（詳細は README.txt 参照）。


## 開発メモ

- 外部ライブラリ不使用。`type="module"` も使っていないため、
  `file://` で直接 `index.html` を開いても動作します。
- ゲームデータ（UMA・アイテム・武器・まほう・職業）は `script.js` 冒頭の
  「2. データ定義」セクションにまとめてあります。新しいUMAや職業を
  追加する際は、基本的にこの配列に要素を追加するだけで済みます。
- 究極ゴリラの出現確率や強さを調整する定数も `script.js` 内にコメント付きで
  記載しています。
- より詳しい仕様・調整値・トラブルシュートは [README.txt](README.txt) を参照してください。
