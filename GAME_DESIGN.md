## 1. ゲームコンセプト

スマホ1画面で延々とプレイできる、**UMA収集ブラウザRPG**。

- 育つのは主人公だけ。**UMAは育成しない。** 捕まえて図鑑に登録する・売ってお金にする「収集・コレクション対象」として扱う。
- ドラクエ風の2Dフィールドを歩き、ランダムエンカウントで戦う「歩く→戦う→強くなる」のループを繰り返す。
- 外部ライブラリ・サーバー不要。HTML/CSS/JavaScriptのみで動作し、GitHub Pagesで配信できる静的サイトとして実装する。
- 操作はスマホ前提(十字キー/スワイプ)。PCでも矢印キーで遊べる。


## 2. 世界観

**[実装済み(最小限)]**

究極のUMA「究極ゴリラ」🦍 を探し求める勇者の子孫たちが、UMA(未確認生物)が
潜む世界を旅する。世界には伝説のUMAも紛れており、低レベルの旅人がいきなり
ラスボス級のUMAに遭遇することもある。

現状はテキスト/絵文字による最小限の世界観のみで、固有の地名・歴史などの
詳細な設定は今後追加予定(詳細未定)。


## 3. ストーリー

**[実装済み]**

- ゲーム開始時、王様からのオープニングメッセージを表示する(初回起動時のみ)。
  「おお、勇者よ……いや勇者の子孫よ。ついにこの時が来た。伝説のUMA
  究極ゴリラを捕まえるのじゃ！」という導入で、プレイヤーに目的を伝える。
  一度見るとセーブデータに記録され、次回以降は表示しない。
- オープニングの直後、初回のみ「目的説明画面」を表示する(§3.5)。
- プレイヤーは「勇者の子孫」。村を出発し、UMAを探す旅に出る。
- 探索中、伝説のUMA「究極ゴリラ」に低確率で遭遇できる。
- **最終目標は「究極ゴリラを捕まえること」。** 通常の「つかまえる」では捕まえられない。
  レベル99・「女神のウクレレ」の所持・HPを1〜10まで削った状態で
  「うたう」コマンドを使うことで捕獲できる(Version 0.6で実装。詳細は §14.5)。
  初遭遇時は専用モーダルを表示し「出会えたこと」を祝う(§14)。
- 究極ゴリラの通常討伐はレベル99・最強装備クラスでも
  ほぼ不可能な難易度にしてある(詳細は §14)。

## 3.8 レベル99到達マイルストーン演出(Version 0.7.1で実装)

**[実装済み]**

プレイヤーが初めてレベル99に到達した戦闘が終わった後、専用のマイルストーンモーダルを表示する。
究極ゴリラ捕獲の直前条件を満たしたことを祝い、次のアクションを明示する。

### 表示タイミング
- 戦闘中のレベルアップでLv99になり、戦闘終了(「OK」ボタン押下)後に表示する。
- `state.pendingLv99 = true` を立てておき、`finishBattle()` 内で判定して開く。
- クリアモーダル(`pendingClear`)との同時発生時は **クリアを優先** し、Lv99演出は出さない。

### 表示内容(`openLv99Modal()`で動的生成)
- 固定テキスト: 「勇者の子孫の力は、限界まで高まった！」
- 女神のウクレレ**未所持**の場合: ウクレレを探すよう案内
- 女神のウクレレ**所持済み**の場合: うたうでクリアできると案内

### 初回のみ表示
- `state.player.level99Shown`(bool)をセーブデータに保存し、一度表示したら再表示しない。
- ニューゲームでリセットされる。
- デバッグ用に「🎖 Lv99演出を再生」ボタン(`?debug=1`)で強制表示できる。

### 設計メモ
- `levelUp()` 内で `p.level === 99 && !p.level99Shown` を判定してフラグを立てる。
- `finishBattle()` でフラグを確認してモーダルを開く(§28エンディングと同じパターン)。


## 4. ゲームループ

**[実装済み]**

1. フィールドを歩く(十字キー/スワイプ/長押し)
2. 草原を歩くとランダムでエンカウント(UMA or 雑魚モンスター)
3. 戦闘: たたかう / まほう / アイテム / つかまえる / にげる
4. 倒す・逃げる・捕まえる、いずれでも経験値を獲得 → レベルアップ
5. レベルアップ時にランダムでまほうを1つ習得
6. フィールドの武器/回復アイテムを取得して強化
7. 商人で売買、神社で転職して立ち回りを調整
8. 1〜7を繰り返しながら、低確率の伝説UMA・究極ゴリラとの遭遇を目指す


## 5. フィールド/マップ

**[実装済み]**

- 1枚のマップ(13列×18行)。ドラクエ風の2D見下ろし、絵文字タイル表示。
- 地形: 草原(エンカウントあり)/村の道(エンカウントなし)/木・水(進入不可)
- 村の中に 🏪 商人、⛩️ 神社(転職)、🍺 酒場(§9.5)を配置。
- マップ上に武器🗡️・回復アイテム🧪が落ちており、触れると自動取得。
- カメラはプレイヤー中心に追従し、マップ端でクランプする。

**[未実装・将来機能]** 複数マップ/ダンジョン/ワールドマップ間の移動。


## 5.6 実家(ホーム)

**[実装済み]**

フィールド上の 🏠 タイル(H)がプレイヤーの「実家」。スタート地点と同じ位置に
あり、旅の途中で戻ると回復・セーブができる。

- 🏠 タイルに触れると「実家に帰ってきた。」イベントが発生し、
  「休む / やめる」の選択肢が表示される。
- **「休む」を選ぶと:**
  - HPとMPを全回復
  - 状態異常(アレルギー/におい等)をすべて治療
  - オートセーブ(「セーブしました。」と表示)
- **「やめる」を選ぶ:** 何もせずにモーダルを閉じる。
- 旅で危なくなったら実家に戻ることで立て直しができる。

### 設計メモ

- フィールドの `movePlayer()` 内で tile==='H' を検知して `openHomeModal()` を呼ぶ。
- 回復はプレイヤーが能動的に「休む」を選んだ時のみ発動(自動ではない)。
- フィールドドロップ案(§5.7参照)が実装された場合、実家付近にセーフゾーンを
  維持する設計を考慮すること。


## 5.7 宝箱システム(Version 0.4.4で実装)

**[実装済み]**

フィールド上に宝箱シンボル(`B` タイル、表示: 🎁)を配置した。
プレイヤーが宝箱に触れると中身を入手できる。

### 宝箱の挙動

- 初回接触: 「🎁 宝箱を開けた！ ○○を手に入れた！」とトーストで通知する。
- 再接触: 「📦 宝箱は空だった…」と表示し、何も入手しない。
- 開封済みの宝箱は `📦` に見た目が変わる。
- 開封状態はセーブデータ(`state.openedChests`)に保存される。

### 宝箱の中身(重み付き乱択)

| 種別 | 内容 | 備考 |
|---|---|---|
| お金 | 5G / 10G / 20G / 50G | 最も出やすい |
| 回復アイテム | コーヒー / パン / お弁当 / せき止めシロップ / デオドラントスプレー | 出やすい |
| 低〜中ランク装備 | ワイヤーブラシ/ノコギリ/サバイバルナイフ/鉄の棒/木製バット/テニスラケット/ロックT/革ジャン/鉄のたて/ヘルメット | 出にくい |

装備はすでに所持済みの場合、代わりに売却額相当のゴールドを獲得する。

### 宝箱の配置(初期: 4箱)

マップ上の草原エリアに4つ配置。全探索のご褒美として機能する。
将来的に箱の数や配置は増やす予定。

### 設計メモ

- `RAW_MAP` 内の `B` 文字が宝箱タイルを表す。
- `state.openedChests["x,y"] = true` で開封済みを管理。
- `movePlayer()` で `tile === "B"` を検知し `openChest(x, y)` を呼ぶ。
- 宝箱タイルはエンカウントが発生しない安全タイル(SAFE_TILE)。
- `CHEST_DROPS` 配列で重み付き抽選し、`weightedPick()` を共用する。


## 5.8 フィールド上でのアイテム使用(Version 0.4.4で実装)

**[実装済み]**

画面上部ステータスバーの「🎒アイテム」ボタンを押すと、
フィールド上(戦闘外)でアイテムを使用できるようになった。

### 使用可能なアイテム

| 種別 | アイテム | 効果 |
|---|---|---|
| 回復 | やくそう | HP+15 |
| 回復 | コーヒー | HP+10 |
| 回復 | パン | HP+20 |
| 回復 | お弁当 | HP+40 |
| 回復 | ラーメン | HP全回復 |
| 治療 | せき止めシロップ | アレルギーを治す |
| 治療 | デオドラントスプレー | においを治す |

### 制約

- HP満タン時は回復アイテムを使えない(「今は使う必要がない。」と表示し消費しない)。
- 対応する状態異常でない場合は治療アイテムを使えない(同上)。
- 戦闘中はボタンを押しても何もしない。
- 使用後はアイテム画面の所持数が即時更新される。
- セーブが自動で実行される。

### 設計メモ

- `openModal("field-item-modal")` でフィールドアイテムモーダルを開く。
- `renderFieldItemBody()` で ITEM_DATA を動的に読み込んでリスト表示。
- `useFieldHealItem(itemId)` / `useFieldCureItem(itemId)` で実際に使用する。
- 使用後は `renderFieldItemBody()` で再描画して所持数を更新する。


## 5.5 横スクロールマップ案(将来の大型改修候補)

**[未実装・将来機能]** ※プロデューサー提案。v0.9以降または専用ブランチで検討。

現在の固定グリッド1枚マップ(上下左右移動)から、将来的に横スクロール型
フィールドへ変更する案。

### 構想(v0.7.1時点の整理)

- **イメージ**: 「ボコスカウォーズ」風のレトロ横スクロール。
- **ステージ構成**: 3面〜5面程度を左右に行き来できる。
  - 面ごとに敵の種類・強さ・宝箱・NPC・施設(商人/神社/酒場/実家)を配置。
  - 面が変わるほど難易度が上がり、奥の面に究極ゴリラが出やすくなる。
- **移動方向**: 主に横方向。縦移動は補助的に残す(または省く)。
- **狙い**: 冒険感・レトロゲーム感の強化。「次の面に進む」達成感。

### 影響範囲(大規模改修)

| 影響箇所 | 現状 | 変更後 |
|---|---|---|
| `RAW_MAP` | 1枚固定グリッド | 複数マップの配列 |
| `renderField()` | 固定ビューポート | 横スクロール対応 |
| カメラ | X/Y クランプ | 横スクロール追従 |
| エンカウント | 草原タイル均一 | 面ごとに敵テーブル切替 |
| セーブ座標 | 未保存 | 現在面+座標をセーブ必須 |
| 施設配置 | 固定タイル | 面ごとのレイアウト |

### 実装方針

- **v0.9以降**または**専用ブランチ**で着手する大型改修として扱う。
- 既存の1枚マップ版は安定しているため、横スクロール版はブランチを切って開発する。
- 現行コードへの互換パッチ方式ではなく、マップ関連モジュールを書き直す想定。


## 6. UMA・モンスターの分類

**[実装済み]**

このゲームは「敵を倒すゲーム」ではなく、**UMAや敵を弱らせる → 捕まえる、
または(力尽きて)逃げられる → 経験値や報酬を得る** というUMA収集RPGである
(§13参照)。これを踏まえ、敵は3つに分類する。すべて `type` フィールドで
区別し、データは `script.js` の `UMA_DATA` / `NON_UMA_DATA` に集約する
(メタル系も実装上は `NON_UMA_DATA` に含め、`type: "metal"` で区別する)。

| 分類 | type | 図鑑/捕獲/売却 | 目的 |
|---|---|---|---|
| UMA | `"uma"` | 対象 | 収集・コレクション(§1) |
| 通常モンスター | `"monster"` | 対象外 | 経験値・お金・アイテム目的の道中の敵 |
| メタル系 | `"metal"` | 対象外 | 経験値稼ぎ用のボーナス的な敵(§6.3) |

### 6.2 特殊行動の仕様

- **アレルギー/におい付与**: 攻撃が命中した時、低確率で状態異常を付与する
  (§13.5と同じ仕組み)。`inflicts: { id, chance, duration }` で定義。
- **MP吸収**: 攻撃時、低確率でプレイヤーのMPを一定値減らす。
  `drainsMp: { chance, amount }` で定義。
- **所持金を盗む**: 攻撃時、低確率でプレイヤーの所持金を一定額減らす。
  `stealsGold: { chance, amount }` で定義。
- **不意打ち(先制攻撃)**: 戦闘開始時、プレイヤーが行動する前に1回だけ
  攻撃してくる。`ambush: true` で定義。
- **素早い**: プレイヤーが「にげる」を選んだ時の成功率(`fleeRate`)を
  通常より低く設定することで表現する。新しい仕組みは追加していない。

### 6.3 メタル系(経験値稼ぎ用のボーナス敵。Version 0.6.1で調整)

**[実装済み]**

防御力が非常に高く通常攻撃はほとんど通らないが、経験値が大きい
「捕まえなくてもよい」ボーナス敵。出現率は低く、`METAL_ENCOUNTER_CHANCE`
(既定0.04)で専用に抽選する(UMAのレア抽選とは別枠)。図鑑・捕獲対象外。

| id | 名前 | 最低Lv | HP | 攻撃力 | 防御 | EXP | 想定タイミング |
|---|---|---|---|---|---|---|---|
| metalgorilla | メタルゴリラ | 1 | 8 | 3 | 25 | 120 | 序盤〜中盤の経験値ボーナス |
| haguremetalgorilla | はぐれメタルゴリラ | 10 | 12 | 5 | 40 | 400 | 中盤〜終盤の経験値ボーナス |
| fullmetalgorilla | フルメタルゴリラ | 20 | 16 | 8 | 60 | 1000 | 終盤向けの高経験値ボーナス |

**Version 0.6.1でEXPを大幅増量**: レベル99到達のテンポ改善のため
メタルゴリラ: 40→120、はぐれ: 120→400、フルメタル: 300→1000 に増量。

## 8.5 装備システム

**[実装済み(所持制・商人購入対応)]**

武器に加えて、防具・盾・兜の装備スロットを追加した。データは
`script.js` の `EQUIP_WEAPON_DATA` / `ARMOR_DATA` / `SHIELD_DATA` /
`HELMET_DATA` に集約。§8のレガシー武器(`WEAPON_DATA`)とは別の配列・別の
ボーナス枠として加算される(レガシー仕様を壊さないため意図的に分離)。

- 各スロット(武器/防具/盾/兜)につき1つ装備できる。
- 各リストの先頭の項目(木の棒/Tシャツ/段ボールのたて/男塾ハチマキ)は

- **Version 0.4.1で所持制に変更。** プレイヤーは `player.ownedWeapons` /
  `ownedArmors` / `ownedShields` / `ownedHelmets`(idの配列)に持っている
  装備だけを装備できる。所持していない装備は装備変更画面で
  「未所持」と表示され、選択できない(ゲームバランス対応)。
  Version 0.4で自由に選べていた状態からの移行(セーブ済みプレイヤー)は、
  読み込み時にその時点で装備していたものを所持品として自動的に
  引き継ぐ(既存プレイヤーが装備を失わないようにするための救済措置)。
- 画面右上「🎽装備」から装備変更画面を開き、所持している項目に
  いつでも切り替えられる。
- ステータス再計算(`recomputeStats()`)は
  `攻撃力 = ベース + 職業補正 + レガシー武器加算 + 装備武器のatkBonus`、
  `防御力 = ベース + 職業補正 + 防具/盾/兜のdefBonusの合計`

**[実装済み]**

## 13. 戦闘

**[実装済み]**

- コマンド: ⚔たたかう / ✨まほう / 🧪アイテム / 🪤つかまえる / 🎵うたう / 💨にげる
- **Version 0.4.2より、「倒す」という表現をやめた。** このゲームは
  「敵を倒すゲーム」ではなく「UMAや敵を弱らせる→捕まえる、または
  逃げられる→経験値や報酬を得る」というUMA収集RPGであるという方針を
  明確にするため、敵のHPが0になっても撃破ではなく
  **「○○に逃げられた！」** という表現にした。結果自体(経験値が入る)は
  変わらず、さらに少額の所持金を獲得するようにした(「○○Gを手に入れた！」)。
  捕獲・逃走(プレイヤー側)の結果には影響しない、HP0到達時の表現のみの変更。
- 倒す(逃げられる)・逃げる・捕まえる、いずれの結果でも経験値を獲得する。
  HPが0になった場合(逃げられた場合)のみ、追加で所持金も獲得する
  (`Math.ceil(敵のEXP / 2)` G)。
- ダメージ計算: `max(1, 攻撃力 + 乱数 - 防御力)` を基本とする。
- **捕獲成功率(Version 0.4.2で強化):**
  `捕獲率ベース + (1-敵HP比率)×HP連動係数 + 職業補正 + アイテム補正 - におい補正`
  を0.05〜0.95にクランプする。HP連動係数は通常の敵が0.50、レアUMA
  (`isRare`)は0.25(例外的に難しくする)。究極ゴリラのみ追加で0.02を
  上限にする(§14)。HPを1〜10程度まで減らすと、通常の敵はかなり
  捕まえやすくなる。
- 逃走成功率: `敵ごとの基本逃走率 + 職業補正` を0.05〜0.97にクランプ。
  「素早い」敵は基本逃走率を低めに設定することで、プレイヤーが
  逃げにくいようにしている(§6.2)。
- HPが0になると「倒れてしまった」表示の上で村に全回復で送還される
  (ペナルティは軽め)。
- **Version 0.4.5より、戦闘終了時に必ず「OK」ボタンで結果を確認してから
  フィールドに戻る。** 戦闘ログに結果(逃げられた/捕まえた/逃げ切った +
  経験値 + 獲得ゴールド)が表示された後、OKボタンを押すと画面が切り替わる。
  これにより、スクロールして結果を見逃す問題を解消した。

状態異常との関係は §13.5 を参照。敵の特殊行動(不意打ち/MP吸収/
所持金を盗む)は §6.2 を参照。

**[未実装・将来機能]** 複数体との同時戦闘、仲間の戦闘参加(§10)。


## 13.7 戦闘開始直後の誤タップ防止(Version 0.4.5で実装)

**[実装済み]**

スマホで十字キーを連打しているとエンカウントが発生し、画面切り替わりの
直後に指の位置にある戦闘コマンドを誤って押してしまう問題があった。

- 戦闘開始から **800ms** の間、すべての戦闘コマンドボタンを `disabled` にする。
- 敵の登場メッセージ(「○○が現れた！」)を読めるようにするための猶予時間。
- 800ms後に `setBattleLocked(false)` + 「どうする？」ログを表示してから
  コマンドを受け付ける。
- 不意打ち(`ambush: true`)の敵の場合は、戦闘開始時から `setBattleLocked(true)`
  にしたまま `enemyTurn()` に入り、敵の攻撃処理の後に `setBattleLocked(false)`
  を呼ぶ(敵の攻撃アニメーションが入力ロック期間を代替する)。


## 13.8 スマホ操作改善(Version 0.4.5で実装)

**[実装済み]**

スマホブラウザで長押しすると出る「コピー/検索/翻訳」メニューや
テキスト選択ハイライトを抑制した。

### CSS対応

- `* { user-select: none; -webkit-user-select: none; -webkit-touch-callout: none; }`
  全要素のテキスト選択・ iOS の「コールアウト」メニューを無効化。
- `button { touch-action: manipulation; }` ボタン全般のダブルタップズームと
  長押しコンテキストメニューを抑制。
  (十字キー `.dpad-btn` の `touch-action: none` はより高い特異度で維持される)

### JavaScript対応

- `#game` 要素の `contextmenu` イベントに `preventDefault()` を追加。
  長押しで出る右クリックメニュー(Androidで多い)を抑制する。

## 14. ラスボス: 究極ゴリラ

  という体験を演出する(倒さなくても発見した時点で目標達成という位置づけ)。

## 14.5 究極ゴリラ捕獲条件

**[実装済み(Version 0.6)]**

通常の「つかまえる」コマンドでは究極ゴリラを捕獲できない
(「究極ゴリラには普通の捕獲は通用しない！」メッセージを表示し、
ロープ使用時も同様にブロックする)。

### 捕獲成功条件
- プレイヤーがレベル99以上
- 「女神のウクレレ」を所持している(`state.player.hasUkulele`)
- 究極ゴリラのHPが1〜10の範囲
- 戦闘コマンド「うたう」を使用する

### 条件未達時のメッセージ
| 条件 | メッセージ |
|---|---|
| レベル不足 | 「歌声は森に響いた……しかし、まだ力が足りない。レベル99になれば届くかもしれない。」 |
| 女神のウクレレなし | 「歌おうとしたが、何かが足りない。伝説の楽器が必要なようだ。」 |
| HP高すぎ(>10) | 「究極ゴリラはまだ荒ぶっている！もっと弱らせなければ歌は届かない。」 |

### 捕獲成功時の演出メッセージ(バトルログ)
1. 「勇者の子孫は女神のウクレレを奏でた。」
2. 「森にやさしいメロディが響きわたる。」
3. 「究極ゴリラの怒りが静まっていく……」
4. 「究極ゴリラは静かに目を閉じた。」
5. 「究極ゴリラを捕まえた！」

### 暫定クリア
- 捕獲成功時に `state.gameCleared = true` を保存(セーブデータに記録)。
- OKボタンでフィールドに戻った後、暫定クリアモーダルを表示する。
- 「しかし、究極ゴリラは森へ帰ることを望んでいる。いつかまた、森の奥で会えるだろう。」
- クリア後もゲームは続けて遊べる(2周目要素は未実装)。

### 女神のウクレレ
- 重要アイテム。消耗しない(使用回数無制限)。売却不可。
- `state.player.hasUkulele`(bool)としてセーブ。
- マップに専用タイル 'U'(🪗)を追加し、一度開けると再入手不可。
- ステータス画面で所持状態を確認できる。

### 将来仕様
- BGM/SE・アニメーション付き本格エンドロール(v0.7で暫定実装済み。§28)
- 2周目要素・クリア後の強化コンテンツ

## §34. BGM/SE・サウンド設定 (Version 0.8.4)

### 概要

Web Audio API を使った軽量 BGM/SE システムを追加する。外部音声ファイル不要。
スマホの自動再生制限に対応し、ユーザーが操作した時のみ音を鳴らす。
サウンド設定はニューゲームでリセットされない（別 localStorage キーで保存）。

### サウンド設定

- 設定キー: `"ultimateGorillaSoundV1"`（`SAVE_KEY` とは別）
- 設定項目: `soundEnabled`（全体）、`bgmEnabled`（BGM）、`seEnabled`（SE）
- デフォルト: すべて false（初回起動時は音なし）
- ニューゲームでリセットしない（プレイヤー環境設定として扱う）

### BGM 種別

| 種別 | タイプ | 説明 |
|---|---|---|
| フィールドBGM | `"field"` | 明るいレトロRPG風ループ。Cメジャー 120BPM |
| バトルBGM | `"battle"` | 緊張感のある速い曲。Aマイナー 150BPM |
| エンディングBGM | `"ending"` | 穏やかなアルペジオ風。Fメジャー 80BPM |

BGM遷移:
- フィールド歩行（dpad/キー入力） → `updateBGM("field")`
- 戦闘開始（actuallyStartBattle） → `updateBGM("battle")`
- 戦闘終了（finishBattle） → `updateBGM("field")`
- エンディング開始（openEndingModal） → `updateBGM("ending")`
- エンディング終了 → `updateBGM("field")`

### SE 種別

| 種別 | タイプ | 発火タイミング |
|---|---|---|
| ボタン | `"btn"` | — |
| バトル開始 | `"battleStart"` | `actuallyStartBattle()` |
| 攻撃 | `"attack"` | `doFight()` |
| ダメージ | `"damage"` | `enemyTurn()` |
| 捕獲成功 | `"captureOk"` | `attemptCapture()` 成功時 |
| 捕獲失敗 | `"captureFail"` | `attemptCapture()` 失敗時 |
| レベルアップ | `"levelUp"` | `levelUp()` |
| 宝箱を開ける | `"chestOpen"` | `openChest()` / `openLegendaryChest*()` / `openUkuleleChest()` |
| アイテム入手 | `"itemGet"` | `giveKingReward()` |
| エンディング開始 | `"endingStart"` | `openEndingModal()` |

### 音量

- BGM 全体: 0.05 〜 0.06（vol フィールド）
- SE 全体: 0.08 〜 0.12（各ノートの vol フィールド）

### 技術仕様

- `AudioContext` / `webkitAudioContext` フォールバック
- OscillatorNode + GainNode の組み合わせで発音
- BGM は短いノート列を `setTimeout` で再スケジュールするループ方式
- SE は単発の短い音バースト（複数ノートを時間差で重ねる）
- `initAudioContext()` はユーザー操作イベントから呼ぶ（iOS自動再生制限対応）
- すべての Audio 処理を try/catch でラップし、失敗時はサイレント無視
- AudioContext が使えない環境でも graceful degradation

### 設定画面 UI

設定モーダルの `renderSettingsBody()` に以下を追加:

```
🔊 サウンド設定
[🔊 サウンド: ON]  [🔇 サウンド: OFF]  ← トグルボタン（現在状態を表示）
[🎵 BGM: ON]       ← soundEnabled=false の時は opacity:0.45
[🔔 SE: ON]        ← soundEnabled=false の時は opacity:0.45
```

### デバッグ機能（`?debug=1`）

- `[TEST] SEを鳴らす` — levelUp SE を再生テスト
- `[TEST] フィールドBGM` — フィールドBGM を強制再生（soundEnabled/bgmEnabled を ON にする）
- `[TEST] バトルBGM` — バトルBGM を強制再生
- `[TEST] エンディングBGM` — エンディングBGM を強制再生
- `[TEST] BGM停止` — BGM を停止
