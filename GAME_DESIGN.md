## 1. ゲームコンセプト

スマホ1画面で延々とプレイできる、**UMA収集ブラウザRPG**。

- 育つのは主人公だけ。**UMAは育成しない。** 捕まえて図鑑に登録する・売ってお金にする「収集・コレクション対象」として扱う。
- ドラクエ風の2Dフィールドを歩き、ランダムエンカウントで戦う「歩く→戦う→強くなる」のループを繰り返す。
- 外部ライブラリ・サーバー不要。HTML/CSS/JavaScriptのみで動作し、GitHub Pagesで配信できる静的サイトとして実装する。
- 操作はスマホ前提(十字キー/スワイプ)。PCでも矢印キーで遊べる。


## 2. 世界観

**[実装済み(最小限)]**

究極のUMA「究極ゴリラ」🦍 を探し求める勇者の子孫たちが、UMA(未確認生物)が
潜む世界を旅する。世界には伝説のUMAも紛れており、低レベルの旅人がいきなり
ラスボス級のUMAに遭遇することもある。

現状はテキスト/絵文字による最小限の世界観のみで、固有の地名・歴史などの
詳細な設定は今後追加予定(詳細未定)。


## 3. ストーリー

**[実装済み]**

- ゲーム開始時、王様からのオープニングメッセージを表示する(初回起動時のみ)。
  「おお、勇者よ……いや勇者の子孫よ。ついにこの時が来た。伝説のUMA
  究極ゴリラを捕まえるのじゃ！」という導入で、プレイヤーに目的を伝える。
  一度見るとセーブデータに記録され、次回以降は表示しない。
- オープニングの直後、初回のみ「目的説明画面」を表示する(§3.5)。
- プレイヤーは「勇者の子孫」。村を出発し、UMAを探す旅に出る。
- 探索中、伝説のUMA「究極ゴリラ」に低確率で遭遇できる。
- **最終目標は「究極ゴリラを発見すること」。** 討伐や捕獲は目的ではなく、
  「出会えたこと」自体がゲームのクリア体験になる(発見時に専用メッセージを表示)。
- 究極ゴリラを倒す・捕まえることは、レベル99・最強装備クラスでも
  ほぼ不可能な難易度にしてある(詳細は §14)。これは「本当に倒す」ことを
  目的にしないという物語上の意図でもある。


## 3.5 目的説明画面

**[実装済み]**

初回プレイ時、ユーザーテストで「何を目指せばいいか分からない」という
フィードバックがあったため追加した。オープニングモーダルを閉じた直後に
1回だけ自動で表示する。

内容:

```
目的:
伝説のUMA「究極ゴリラ」を捕まえよう！

そのためにやること:
・フィールドを歩いてUMAを探す
・UMAを倒す、逃げる、捕まえることで経験値を得る
・レベルを上げる
・装備を集める
・UMA図鑑を埋める
・最終的に究極ゴリラに挑む
```

「OK」ボタンで閉じる。一度見るとセーブデータに記録され(`seenGoal`)、
自動表示は二度目以降出ないが、⚙️設定モーダルの「🎯目的を見る」ボタンから
いつでも再表示できる。


## 4. ゲームループ

**[実装済み]**

1. フィールドを歩く(十字キー/スワイプ/長押し)
2. 草原を歩くとランダムでエンカウント(UMA or 雑魚モンスター)
3. 戦闘: たたかう / まほう / アイテム / つかまえる / にげる
4. 倒す・逃げる・捕まえる、いずれでも経験値を獲得 → レベルアップ
5. レベルアップ時にランダムでまほうを1つ習得
6. フィールドの武器/回復アイテムを取得して強化
7. 商人で売買、神社で転職して立ち回りを調整
8. 1〜7を繰り返しながら、低確率の伝説UMA・究極ゴリラとの遭遇を目指す


## 5. フィールド/マップ

**[実装済み]**

- 1枚のマップ(13列×18行)。ドラクエ風の2D見下ろし、絵文字タイル表示。
- 地形: 草原(エンカウントあり)/村の道(エンカウントなし)/木・水(進入不可)
- 村の中に 🏪 商人、⛩️ 神社(転職)、🍺 酒場(§9.5)を配置。
- マップ上に武器🗡️・回復アイテム🧪が落ちており、触れると自動取得。
- カメラはプレイヤー中心に追従し、マップ端でクランプする。

**[未実装・将来機能]** 複数マップ/ダンジョン/ワールドマップ間の移動。

現在の上下左右に動ける1枚マップから、将来的には横スクロール型の

## 6. UMA・モンスターの分類

**[実装済み]**

このゲームは「敵を倒すゲーム」ではなく、**UMAや敵を弱らせる → 捕まえる、
または(力尽きて)逃げられる → 経験値や報酬を得る** というUMA収集RPGである
(§13参照)。これを踏まえ、敵は3つに分類する。すべて `type` フィールドで
区別し、データは `script.js` の `UMA_DATA` / `NON_UMA_DATA` に集約する
(メタル系も実装上は `NON_UMA_DATA` に含め、`type: "metal"` で区別する)。

| 分類 | type | 図鑑/捕獲/売却 | 目的 |
|---|---|---|---|
| UMA | `"uma"` | 対象 | 収集・コレクション(§1) |
| 通常モンスター | `"monster"` | 対象外 | 経験値・お金・アイテム目的の道中の敵 |
| メタル系 | `"metal"` | 対象外 | 経験値稼ぎ用のボーナス的な敵(§6.3) |

### 6.2 特殊行動の仕様

- **アレルギー/におい付与**: 攻撃が命中した時、低確率で状態異常を付与する
  (§13.5と同じ仕組み)。`inflicts: { id, chance, duration }` で定義。
- **MP吸収**: 攻撃時、低確率でプレイヤーのMPを一定値減らす。
  `drainsMp: { chance, amount }` で定義。
- **所持金を盗む**: 攻撃時、低確率でプレイヤーの所持金を一定額減らす。
  `stealsGold: { chance, amount }` で定義。
- **不意打ち(先制攻撃)**: 戦闘開始時、プレイヤーが行動する前に1回だけ
  攻撃してくる。`ambush: true` で定義。
- **素早い**: プレイヤーが「にげる」を選んだ時の成功率(`fleeRate`)を
  通常より低く設定することで表現する。新しい仕組みは追加していない。

(既定0.04)で専用に抽選する(UMAのレア抽選とは別枠)。図鑑・捕獲対象外。

## 7. 図鑑

**[実装済み]**

UMAごとに3段階の状態を持つ。売却してもこの記録は消えない(「捕まえた実績」)。

- **未発見**: 一度も遭遇していない(「？？？」表示)
- **発見済み**: 戦闘で遭遇したが、まだ捕獲に成功していない
- **捕獲済み**: 一度でも捕獲に成功した

画面右上「📖図鑑」ボタンから一覧モーダルを開く。

## 8.5 装備システム

**[実装済み(所持制・商人購入対応)]**

武器に加えて、防具・盾・兜の装備スロットを追加した。データは
`script.js` の `EQUIP_WEAPON_DATA` / `ARMOR_DATA` / `SHIELD_DATA` /
`HELMET_DATA` に集約。§8のレガシー武器(`WEAPON_DATA`)とは別の配列・別の
ボーナス枠として加算される(レガシー仕様を壊さないため意図的に分離)。

- 各スロット(武器/防具/盾/兜)につき1つ装備できる。
- 各リストの先頭の項目(木の棒/Tシャツ/段ボールのたて/男塾ハチマキ)は
  ボーナス0の初期装備で、最初から所持・装備している。
- **Version 0.4.1で所持制に変更。** プレイヤーは `player.ownedWeapons` /
  `ownedArmors` / `ownedShields` / `ownedHelmets`(idの配列)に持っている
  装備だけを装備できる。所持していない装備は装備変更画面で
  「未所持」と表示され、選択できない(ゲームバランス対応)。
  Version 0.4で自由に選べていた状態からの移行(セーブ済みプレイヤー)は、
  読み込み時にその時点で装備していたものを所持品として自動的に
  引き継ぐ(既存プレイヤーが装備を失わないようにするための救済措置)。
- 画面右上「🎽装備」から装備変更画面を開き、所持している項目に
  いつでも切り替えられる。
- ステータス再計算(`recomputeStats()`)は
  `攻撃力 = ベース + 職業補正 + レガシー武器加算 + 装備武器のatkBonus`、
  `防御力 = ベース + 職業補正 + 防具/盾/兜のdefBonusの合計`
  に拡張した。武器の `mpBonus` や防具の `hpBonus` がある場合はMP/HPの
  上限にも加算される。

## 9. 商人

**[実装済み]**

## 10. 仲間システム

**[未実装・将来機能]** ※プロデューサー提案。`TODO.md` の Version 0.5 に登録済み。

### 仕様

- フィールド上を歩いているNPCが存在する。
- NPCは「シンボルエンカウント」方式(ランダムではなく、フィールド上に
  見えているNPCに接触する/近づくことで発生する)。
- NPCに話しかけることができる。
- 話しかけた結果は以下のいずれかになる:
  1. **仲間になる**
  2. **断られる**
  3. **戦闘になる**
- 仲間になった場合:
  - 主人公と一緒にフィールドを歩く(追従)。
  - パーティーメンバーになる。
  - 仲間ごとに固有の能力(スキル/補正)を持つ。

### 設計メモ(実装時に検討する項目)

- NPCの移動パターン(固定/巡回/プレイヤー追従なし)をどう実装するか。
- 「シンボルエンカウント」は既存のランダムエンカウント(歩数×確率)とは
  別の仕組みになる。マップ上に常時存在するオブジェクトとして
  `state.items` とは別の「NPC配列」を新設し、接触判定を追加する想定。
- 仲間の人数上限、パーティー編成UI、仲間の固有能力が戦闘にどう介入するか
  (例: 自動で追加行動する/パッシブ効果のみ等)は未定。
- 「断られる」「戦闘になる」の判定基準(NPCごとの性格・プレイヤーレベル等)も未定。
- UMAとは異なり、仲間は「育成しない」方針(§1)の対象外にするかどうかも
  プロデューサーと要確認(現状の方針では主人公以外は育成しない想定だが、
  仲間固有能力の扱い次第で例外にする可能性がある)。

## 13. 戦闘

**[実装済み]**

- コマンド: ⚔たたかう / ✨まほう / 🧪アイテム / 🪤つかまえる / 💨にげる
- **Version 0.4.2より、「倒す」という表現をやめた。** このゲームは
  「敵を倒すゲーム」ではなく「UMAや敵を弱らせる→捕まえる、または
  逃げられる→経験値や報酬を得る」というUMA収集RPGであるという方針を
  明確にするため、敵のHPが0になっても撃破ではなく
  **「○○に逃げられた！」** という表現にした。結果自体(経験値が入る)は
  変わらず、さらに少額の所持金を獲得するようにした(「○○Gを手に入れた！」)。
  捕獲・逃走(プレイヤー側)の結果には影響しない、HP0到達時の表現のみの変更。
- 倒す(逃げられる)・逃げる・捕まえる、いずれの結果でも経験値を獲得する。
  HPが0になった場合(逃げられた場合)のみ、追加で所持金も獲得する
  (`Math.ceil(敵のEXP / 2)` G)。
- ダメージ計算: `max(1, 攻撃力 + 乱数 - 防御力)` を基本とする。
- **捕獲成功率(Version 0.4.2で強化):**
  `捕獲率ベース + (1-敵HP比率)×HP連動係数 + 職業補正 + アイテム補正 - におい補正`
  を0.05〜0.95にクランプする。HP連動係数は通常の敵が0.50、レアUMA
  (`isRare`)は0.25(例外的に難しくする)。究極ゴリラのみ追加で0.02を
  上限にする(§14)。HPを1〜10程度まで減らすと、通常の敵はかなり
  捕まえやすくなる。
- 逃走成功率: `敵ごとの基本逃走率 + 職業補正` を0.05〜0.97にクランプ。
  「素早い」敵は基本逃走率を低めに設定することで、プレイヤーが
  逃げにくいようにしている(§6.2)。
- HPが0になると「倒れてしまった」表示の上で村に全回復で送還される
  (ペナルティは軽め)。

状態異常との関係は §13.5 を参照。敵の特殊行動(不意打ち/MP吸収/
所持金を盗む)は §6.2 を参照。

**[未実装・将来機能]** 複数体との同時戦闘、仲間の戦闘参加(§10)。

## 14. ラスボス: 究極ゴリラ

**[実装済み]**

- ゲームにおける唯一の「ラスボス級」UMA。レベルに関係なく低確率で出現する。
- HP 5000 / 攻撃力150 / 防御60。レベル99・最強装備クラスでも
  通常プレイでは現実的に勝てない強さに調整している。
- 攻撃は毎ターン「かいしんのいちげき」級の特大ダメージ(防御力を
  ほとんど無視する特殊計算)。
- 捕獲成功率は通常の捕獲式の結果に関わらず常に2%を上限にする
  (HPを減らしても上がりやすくしない)。
- 逃走成功率は0.95と高めに調整し、「出会ったら逃げて生き延びる」を
  前提にしたバランスにしている。
- 初遭遇時は専用の発見モーダルを表示し、「ゲームの最終目標を達成した」
  という体験を演出する(倒さなくても発見した時点で目標達成という位置づけ)。

## 14.5 究極ゴリラ捕獲条件(将来仕様。今回は未実装)

**[未実装・将来機能]** ※プロデューサー提案。

通常の「つかまえる」コマンドでは究極ゴリラを捕獲できない
(§14のとおり捕獲率は常に2%以下)。代わりに、以下の専用条件を
満たした場合のみ捕獲できる案。

- プレイヤーがレベル99に到達している
- 最終必須アイテム「女神のウクレレ」を入手している
- 戦闘コマンドに「うたう」が追加されており、それを使用する
- 究極ゴリラのHPを1〜10程度まで減らしている

これらすべてを満たした状態で「うたう」を使用すると、究極ゴリラを
捕獲できる。捕獲後のエンディングでは、究極ゴリラは森へ帰る
(原作のクリア後演出として、捕獲はゴールであり「飼う」ことが目的では
ないという物語上の意図)。

### 設計メモ(実装時に検討する項目)

- 「女神のウクレレ」の入手方法(購入/ドロップ/イベント報酬等)は未定。
- 「うたう」コマンドは戦闘メニューのどこに追加するか(まほうの一種か、
  専用ボタンか)は未定。
- レベル99到達自体も現状特別な演出がない(§17「レベル99イベント」参照)。
